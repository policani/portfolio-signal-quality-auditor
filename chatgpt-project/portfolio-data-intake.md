# Portfolio Data Intake

## Intake questions

Use these only when the answer is not already present:

1. What source produced the portfolio data?
2. What portfolio, business unit, product area, or program does it cover?
3. What is the data extract date?
4. What status values are currently used?
5. Which fields are expected to identify owner, sponsor, target date, readiness, and KPI?
6. Is the output for cleanup, executive review, intake, prioritization, or operational follow-through?

## Minimum viable intake record

| Field | Required? | Why it matters |
|---|---:|---|
| Initiative name | Yes | Needed for issue tracking and duplicate review. |
| Description or intended outcome | Preferred | Helps distinguish duplicates from related work. |
| Status | Preferred | Needed for status consistency and stale-status checks. |
| Owner | Preferred | Needed for accountability. |
| Sponsor | Preferred | Needed for decision rights and escalation. |
| Last updated date | Preferred | Needed for staleness review. |
| Target date or milestone | Preferred | Needed for aging and deadline risk. |
| Readiness or phase | Preferred | Needed for downstream routing. |
| KPI or success metric | Optional | Needed for KPI hygiene review. |

## Intake output

Before auditing, state:

- Source type
- Row count or record count
- Fields available
- Fields missing
- Screens that can be run with confidence
- Screens that require caution or cannot be run
