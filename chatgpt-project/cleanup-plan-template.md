# Cleanup Plan Template

## Cleanup plan structure

Use this structure after running the audit.

### 1. Executive cleanup summary

- Overall trust level
- Top reporting risks
- Number of records reviewed
- Number of critical, high, medium, and low findings
- Records blocked from downstream use
- Decisions required

### 2. Issue register

| ID | Record | Issue type | Severity | Evidence | Cleanup action | Owner needed | Due / cadence | Human decision |
|---|---|---|---|---|---|---|---|---|

### 3. Cleanup waves

- **Wave 1: Decision blockers** — missing sponsor, stale active work, conflicting status, high-risk duplicates.
- **Wave 2: Governance hygiene** — missing owners, readiness ambiguity, weak next steps, missing target dates.
- **Wave 3: Reporting improvement** — KPI definitions, baselines, cadence, data source clarity.

### 4. Downstream readiness

| Destination | Records eligible | Records blocked | Reason blocked | Owner action needed |
|---|---:|---:|---|---|

### 5. Follow-through packet

Send to PMO Governance Operations Log:

- Cleanup action
- Action owner
- Due date or review cadence
- Decision needed
- Escalation path
- Evidence needed
